**QuickTrips** ha optado por utilizar **MongoDB** como su motor de base de datos debido a sus características destacadas: flexibilidad, alto rendimiento y escalabilidad. MongoDB es una base de datos NoSQL que permite almacenar datos en formato de documentos JSON. Esta estructura facilita el manejo de información diversa y dinámica, ideal para una aplicación como QuickTrips, donde se gestionan datos variados como perfiles de usuarios, reservas y transacciones, sin la necesidad de un esquema rígido y predefinido.

## Colecciones

En la base de datos de QuickTrips, los datos se organizan en tres categorías principales: **Colecciones de Catálogo**, **Colecciones Principales** y **Esquemas Incrustados**. A continuación, se describen estas categorías en detalle.

### Colecciones de Catálogo

Las **colecciones de catálogo** contienen información de referencia que no se modifica directamente mediante la interacción del usuario. Estas entidades se gestionan de forma estática en la base de datos y se utilizan principalmente como fuentes de datos para consultas. Existen para proporcionar valores predeterminados o listas de opciones que se utilizan en otras partes de la aplicación. Ejemplo:

- **Tipo de usuario**: Define los distintos tipos de usuarios en la plataforma y sus respectivos permisos.

### Colecciones Principales

Las **colecciones principales** son el núcleo de la base de datos y representan los elementos más críticos en términos de funcionalidad y operaciones de negocio. Estas entidades son gestionadas activamente por los usuarios a través de la interfaz de la aplicación y contienen datos que se actualizan regularmente. Ejemplos:

- **Usuarios**: Es una lista que contiene esquemas de tipo **Usuario**. El esquema usuario incluye información personal, datos de acceso, entre otros.

### Esquemas Incrustados (Schemas)

Los **esquemas incrustados** complementan a las entidades principales al almacenar información adicional que enriquece los datos sin tener un propósito independiente. Estas estructuras anidadas están diseñadas para proporcionar detalles relacionados con las entidades principales y facilitar su gestión sin necesidad de crear colecciones separadas. Ejemplo:

- **Dirección**: Almacena detalles como la calle, colonia, número, estado y país de los usuarios o ubicaciones asociadas.
