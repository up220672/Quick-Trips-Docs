### Campos usados en base al [rol de usuario](/docs/database/catalogs/user-type)

Todos los roles de usuario utilizan el mismo modelo de datos, pero los campos pertenecientes a otro tip1o de usuario serán **nulos**. Por ejemplo, un usuario con rol 2 (Guest) tendrá el campo `tax_document` y `properties` pero serán `nulos`.

#### Rol 3000: Super Admin

**Usuario Maestro**, es un usuario que se crea directamente en la base de datos, no interactua en el sistema con otros usuarios, al iniciar sesión con este usuario inmediatamente navega a una pantalla especial para añadir o eliminar usuarios de rol 0.
Solo necesita:

- `_id`
- `username`
- `password_hash`
- `role`

#### Rol 0: Admin

Estos usuarios se utilizan para administrar la plataforma. Por lo tanto, no se requiere información adicional. Sin embargo, para mantener la consistencia, se utilizan solo los siguientes campos:

- `_id`
- `username`
- `first_name`
- `last_name`
- `description`
- `email`
- `password_hash`
- `role`
- `profile_picture`
- `language`
- `created_at`
- `last_login`
- `is_active`


#### Rol 2: Guest

**Los mismos campos que el rol 2, más:**

- `notifications`


### Campos  

| Field                          | Type       | Descripción                                                                                  |
|--------------------------------|------------|----------------------------------------------------------------------------------------------|
| `_id`                          | `ObjectId` | Identificador único del usuario.                                                             |
| `username`                     | `String`   | Nombre de usuario (único).                                                                   |
| `first_name`                   | `String`   | Nombre del usuario.                                                                          |
| `last_name`                    | `String`   | Apellido del usuario.                                                                        |
| `description`                  | `String`   | Descripción del usuario                                                                      |
| `email`                        | `String`   | Correo electrónico del usuario (único).                                                      |
| `phone`                        | `String`   | Número de teléfono del usuario.                                                              |
| `password_hash`                | `String`   | Contraseña cifrada del usuario.                                                              |
| `role`                         | `Number`   | Rol del usuario  Ver [User Type](/docs/database/catalogs/user-type).                         |
| `profile_picture`              | `String`   | URL de la foto de perfil del usuario.                                                        |
| `date_of_birth`                | `Date`     | Fecha de nacimiento del usuario.                                                             |
| `language`                     | `String`   | Idioma preferido del usuario (ejemplo: "es", "en").                                          |
| `address`                      | `Object`   | Dirección del usuario  Ver [Address](/docs/database/models/secondary-models/address).        |
| `currency`                     | `String`   | Código de la moneda preferida del usuario (ejemplo: "MXN", "USD").                           |
| `created_at`                   | `Date`     | Fecha de creación del usuario.                                                               |
| `last_login`                   | `Date`     | Fecha del último inicio de sesión del usuario.                                               |
| `is_active`                    | `Boolean`  | Indica si la cuenta del usuario está activa.                                                 |
| `identity_document`            | `String`   | URL del documento de identidad del usuario.                                                  |
| `is_identity_document_verified`| `Boolean`  | Indica si el documento de identidad del usuario ha sido verificado.                         |
| `tax_document`                 | `String`   | URL del documento fiscal (RFC) del usuario.                                                  |
| `is_tax_document_verified`     | `Boolean`  | Indica si el documento fiscal del usuario ha sido verificado.                                |
| `proof_of_residence`           | `String`   | URL del comprobante de domicilio del usuario.                                                |
| `is_proof_of_residence_verified`| `Boolean` | Indica si el comprobante de domicilio del usuario ha sido verificado.                        |
| `criminal_record_certificate`  | `String`   | URL del certificado de antecedentes no penales del usuario.                                  |
| `is_criminal_record_certificate_verified`| `Boolean` | Indica si el certificado de antecedentes no penales del usuario ha sido verificado.    |
| `properties`                   | `Array`    | Arreglo de identificadores de las propiedades asociadas al usuario. Ver [Property](/docs/database/models/property)                        |
| `notifications`                | `Array`    | Arreglo de notificaciones asociadas al usuario. Ver [Notification](/docs/database/models/secondary-models/notification)                        |
