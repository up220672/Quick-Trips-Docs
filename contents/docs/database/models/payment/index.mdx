# Payment Model

El modelo `Payment` está diseñado para almacenar información detallada sobre las transacciones realizadas por los usuarios en la plataforma. Este documento proporciona una guía completa sobre la estructura y el uso del modelo `Payment`.

## Schema

El esquema `Payment` se define utilizando Mongoose, una herramienta de modelado de objetos MongoDB para Node.js. A continuación se muestra la definición del esquema:

```javascript
const mongoose = require('mongoose');
const { AmountDetails } = require('./secondary_models/amountDetails');
const { CardPaymentMethod } = require('./secondary_models/cardPaymentMethod');
const AmountDetailsSchema = require('./secondary_models/amountDetails').AmountDetailsSchema;
const CardPaymentMethodSchema = require('./secondary_models/cardPaymentMethod').CardPaymentMethodSchema;

const PaymentSchema = new mongoose.Schema({
  amount_details: { 
    type: AmountDetailsSchema,
    ref: 'AmountDetails',
    required: false,
    default: AmountDetails(),
  },
  card: { 
    type: CardPaymentMethodSchema, 
    ref: 'CardPaymentMethod',
    required: false,
    default: CardPaymentMethod(),
  },
  current_status: { 
    type: Number, 
    required: false,
    default: null 
  },
  created_at: { 
    type: Date, 
    required: false,
    default: Date.now 
  },
  confirmed_at: { 
    type: Date, 
    required: false,
    default: null 
  }
}, { versionKey: false });

const Payment = mongoose.model('Payment', PaymentSchema);

module.exports = { Payment, PaymentSchema };
```

## Campos del Modelo

| Campo               | Tipo               | Descripción                                                                 |
|---------------------|--------------------|-----------------------------------------------------------------------------|
| `_id`               | `ObjectId`         | Identificador único del documento de pago.                                  |
| `amount_details`    | `AmountDetailsSchema` | Detalles del monto asociado al pago. Ver [Amount Details](/docs/database/models/secondary-models/amount-details). |
| `card`              | `CardPaymentMethodSchema` | Método de pago con tarjeta asociado al pago. Ver [Card Payment Method](/docs/database/models/secondary-models/card-payment-method). |
| `current_status`    | `Number`           | Estado actual del pago. Ver [Payment Status Type](/docs/database/catalogs/payment-status-type). |
| `created_at`        | `Date`             | Fecha y hora de creación del pago.                                          |
| `confirmed_at`      | `Date`             | Fecha y hora de confirmación del pago.                                      |


`amount_details`

Este campo contiene los detalles del monto asociado al pago, incluyendo el subtotal, la comisión de la plataforma, los impuestos y el total. También se especifica la moneda utilizada en la transacción. Para más detalles, consulta el modelo [Amount Details](/docs/database/models/secondary-models/amount-details).

`card`

Este campo almacena la información del método de pago con tarjeta, incluyendo el nombre del titular, los últimos cuatro dígitos de la tarjeta, el tipo de tarjeta, la fecha de expiración y la dirección asociada. Para más detalles, consulta el modelo [Card Payment Method](/docs/database/models/secondary-models/card-payment-method).

`current_status`

Este campo indica el estado actual del pago. Los valores posibles están definidos en el catálogo [Payment Status Type](/docs/database/catalogs/payment-status-type).

`created_at`

Este campo registra la fecha y hora en que se creó el pago.

`confirmed_at`

Este campo registra la fecha y hora en que se confirmó el pago. Si el pago aún no ha sido confirmado, este campo será `null`.
